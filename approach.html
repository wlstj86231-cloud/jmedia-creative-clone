APPROACH

제작은 화면이 아니라 질문에서 시작합니다.

무엇을 만들지보다 왜 만들어야 하는지, 누가 들어오는지, 사용자가 어디로 이동해야 하는지를 먼저 정리합니다.

01목적 진단

브랜드 소개, 정보 허브, 도구형 서비스, 지도형 데이터, 참여형 전환 중 어디에 가까운지 먼저 나눕니다.

02구조 설계

메뉴, URL, 상세 페이지, CTA, 관련 콘텐츠 흐름을 먼저 그립니다.

03화면 제작

정리된 구조 위에 반응형 레이아웃, 인터랙션, 콘텐츠 표현 방식을 얹습니다.

04배포 준비

GitHub, Cloudflare Pages, 도메인 연결, 기본 점검까지 운영 가능한 상태로 정리합니다.

05인계와 확인

완료 후 고객이 확인해야 할 항목과 운영 시 주의할 부분을 문서로 남깁니다.

WORK RULE

범위가 명확해야 결과가 흔들리지 않습니다.

기능 추가, 외부 서비스 연동, 계정 생성, 도메인 연결처럼 작업 범위가 달라지는 부분은 사전에 분리해서 확인합니다. 이 기준이 있어야 일정과 결과물이 같이 안정됩니다.

작업 전에 구조를 정리하면, 제작 중에 덜 흔들립니다.
